TAO Reconstruction Speedup — PRD v2

1. Context

TAO reconstruction uses RecQMLEAlg (QMLE = Quasi-Maximum Likelihood Estimation), which is structurally very similar to JUNO's OMILREC:

AspectJUNO OMILRECTAO RecQMLEAlg
Detector channels~17,613 large PMTs8,048 SiPM channels (4,024 SiPMs × 2)
Fit methodMinuit2 (QMLE → TMLE → QTMLE → Energy)Minuit2 "Simplex" (QMLE only, 4 params: Evis, r, θ, φ)
FCNCalculate_EVLikelihood (~470 calls/evt)QMLERec::QMLE(Evis, r, θ, φ)
Charge modelPoisson + charge PDF convolution (SPE model)Simple Poisson: P(k|λ) = λᵏ e⁻λ / k!
Expected PEnPE map (2D interpolation) × lPDE / LPMTCalibEnergyTH3F::Interpolate(θ_sipm, r, vθ) via ChargeTemplate
Dark noisePer-PMT from calibration DBPer-channel DCR × readout window
CalibrationOMILRECCalibSvc (nPE maps, time/charge PDFs)CalibSvc (DCR, rPDE, bad channels) + CondDB
Baseline envJUNOSW J26.1.1JUNOSW J26.1.0 + TAOSW overlay
Run commandtut_rtraw2rec.pyRecQMLEAlg/share/run.py

The TAO FCN (QMLERec::QMLE) is simpler than OMILREC — pure Poisson likelihood, no SPE convolution, no time fit, no multi-phase. But the core structure is identical: Minuit calls a likelihood function that loops over all channels.

Reference project: omilrecv2 — JUNO OMILREC was optimized from 1525 → 189 ms/evt (8.1x).

2. Goal

Achieve ≥1.5x single-thread speedup on TAO RecQMLEAlg while maintaining reconstruction quality (resolution, energy scale).

3. Architecture Insight

The hotspot is clear from code inspection:

QMLERec::QMLE(Evis, r, θ, φ)           // Called ~N times by Minuit per event
  ├── for i in 0..8047:                 // 8048 channels
  │     ├── v_vec.Angle(channel_vec[i]) // acos of dot product
  │     ├── CalExpChargeHit(r, angle, vθ, Evis)
  │     │     └── charge_template→CalExpChargeHit(r, θ_sipm, vθ)
  │     │           └── TH3F::Interpolate(θ_sipm, r, vθ)  // ROOT histogram lookup
  │     ├── exp_hit *= rPDE[i]
  │     ├── exp_hit += dcr[i]
  │     ├── Poisson: exp(-λ) + k·log(λ) - LnGamma(k+1)
  │     └── m_Likelihood += -2·log(Pro_Qi)
  └── return m_Likelihood

This is essentially the same pattern as omilrecv2's FCN: a Minuit-minimized loop over all channels, dominated by geometry (acos), histogram interpolation (TH3F::Interpolate), and math functions (exp, log, LnGamma).

4. The Integrated Plan: Test Bed + Refactor → FCN Unit Test

Instead of treating "test bed" and "refactor" as separate sequential phases, we merge them into one goal: get a fast, standalone FCN unit test working. This gives us both the test infrastructure and the refactored code structure in one pass.

4.2 A2: Extract FCN as Free Function

The current FCN is QMLERec::QMLE(double Evis, double vr, double vtheta, double vphi) — a member function that accesses this→channel_vec, this→dcr_list, this→rPDE_list, this→fChannelHit, this→bad_channel_list, this→charge_template_ge68.

We extract it as:

namespace tao_qmle {

struct FCNInput {
    int n_channels;                          // 8048
    const TVector3* channel_positions;       // [n_channels]
    const double* hit_charges;               // [n_channels] — fChannelHit
    const float* dcr;                        // [n_channels] — dcr_list
    const float* rpde;                       // [n_channels] — rPDE_list
    const bool* bad_channel;                 // [n_channels]
    ChargeTemplateData* charge_template;     // extracted 3D grid data (no ROOT TH3F*)
    double saturation;                       // 1e5
    double y0;                               // 4500
    double ge_evis;                          // 0.9250
};

double compute_likelihood(double Evis, double r, double theta, double phi,
                          const FCNInput& input);

} // namespace tao_qmle

Key refactoring moves:

  1. Extract TH3F raw grid data (bin edges + content array) from ChargeTemplate — replace TH3F::Interpolate with direct arithmetic on raw arrays (just like omilrecv2 did for ROOT histograms).
  2. Hoist channel_vec[i] geometry — precompute channel_x/y/z as flat double arrays (SoA).
  3. Remove all this→ member accesses — everything passes through FCNInput.
  4. The class member QMLERec::QMLE becomes a thin wrapper: populate FCNInput, call free function.

4.3 A3: Collect FCN Input Fixtures

Add a data dump mode to the algorithm. When enabled, at the first Minuit call of each event, serialize all FCN inputs to binary files:

fixtures/
├── geometry.bin        # channel positions (8048 × 3 doubles)
├── config.bin          # saturation, y0, ge_evis, threshold, etc.
├── template.bin        # TH3F grid: bin edges + content (extracted from ChargeTemplate)
├── evt_0_input.bin     # hit_charges[8048], dcr[8048], rpde[8048], bad_channel[8048]
├── evt_1_input.bin
├── ...
└── golden.csv          # evt_id, Evis_init, r_init, θ_init, φ_init, golden_LL

Collection procedure:

  1. Add --dump-fcn-fixtures true property to QMLERec
  2. In VertexMinimize(), before first Minimize() call: dump FCNInput to TEMP/fixtures/evt_{evtID}_input.bin
  3. Also record the initial parameter values and the LL at that point
  4. Dump geometry, config, template once in initialize()

4.4 A4: Collect FCN Golden Output

For each event fixture, run the reference (unmodified) FCN at the initial point and record the LL value. This is the golden value that all future versions must reproduce.

We also record golden LL at a few different (Evis, r, θ, φ) points per event — not just the starting point — to better constrain the function.

4.5 A5: FCN Unit Test

// tests/unit/test_fcn.cc
#include "doctest.h"
#include "tao_qmle_fcn.h"

TEST_CASE("FCN reproduces golden LL values") {
    tao_qmle::FCNInput input;
    load_fixture("fixtures/geometry.bin", ...);
    load_fixture("fixtures/config.bin", ...);
    load_fixture("fixtures/template.bin", ...);
    load_golden("fixtures/golden.csv");

    for (auto& row : golden) {
        load_event_fixture("fixtures/evt_" + row.evt_id + "_input.bin", input);
        double ll = tao_qmle::compute_likelihood(row.Evis, row.r, row.theta, row.phi, input);
        double rel_err = abs(ll - row.golden_ll) / abs(row.golden_ll);
        CHECK(rel_err <= 1e-13);
    }
}

This test runs in milliseconds (no SNiPER, no Minuit, no ROOT file I/O after loading), making it a fast pre-commit gate.

9. Key Difference from OMILREC

TAO's FCN is simpler than OMILREC in one critical way: it uses a pure Poisson likelihood, not the SPE-convolution model. This means:

  • No charge PDF histogram interpolation → one less ROOT dependency to extract
  • No Poisson k-loop (k=1..100) → much less compute per channel
  • LnGamma(k+1) can be precomputed as a lookup table (k is integer, bounded)
  • The k=0 (unfired) case simplifies to exp(-λ) — algebraic optimization possible

This also means the relative gains from each optimization category may differ from omilrecv2. The TH3F interpolation and geometry (acos) likely dominate more, while the Poisson math overhead is smaller.
