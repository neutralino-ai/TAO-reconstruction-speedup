# TAO Reconstruction Speedup — PRD

## 1. Context
TAO reconstruction via TAOSW. Reference: omilrecv2 (JUNO OMILREC, 1525→189 ms/evt, 8.1x).

## 2. Goal
≥1.5x single-thread speedup, no resolution degradation.

## 3. Phase Overview

| Step | What | Deliverable |
|------|------|-------------|
| **1**: Test bed | Build + extract FCN + fixture + 2 tests | E2E test + FCN unit test both pass |
| **2**: Profile | perf/flamegraph/phase-timing | Hotspot ranking |
| **3**: Optimize | One commit per opt, measure speed+drift | Speed improves, both tests still pass |
| **4**: Benchmark | 100-event timing, speed.csv | ms/evt per commit |
| **5**: Iterate | 2→3→4 loop | ≥1.5x reached |
| **6**: Acceptance | Resolution vs pimin baseline | No degradation |

---

## 4. Step 1: Test Bed

```
1.1 Build              → libRecQMLEAlg.so + libQMLEFCN.a
1.2 Refactor           → extract QMLERec::QMLE() as free function ComputeQMLE()
1.3 E2E test           → ESD → reco vertex (x,y,z,E)       — test = test_consistency.py
1.4 FCN fixture        → capture ALL FCN inputs + output from pimin baseline
1.5 FCN unit test      → load fixture, call ComputeQMLE(), compare to golden
```

1.2 must complete before 1.4 (fixture uses the free function).
1.4 must complete before 1.5 (unit test needs the fixture).
1.3 and 1.5 together are the test-bed deliverables: two independent tests covering full pipeline and FCN-only.

### 1.1 Build
Build `libRecQMLEAlg.so` and `libQMLEFCN.a` against CVMFS J26.1.0 + pimin's TAOSW overlay.
**Status**: ✅

### 1.2 Refactor — Extract FCN as Free Function
Extract `QMLERec::QMLE()` → `ComputeQMLE(input, Evis, vr, vtheta, vphi)`.
**Input struct** `QMLEInput` must contain **everything** the FCN needs — not just hits:
- `fChannelHit[8048]` — calibrated charge per channel
- `channels[8048]` — per-channel: position (x/y/z), rPDE, dcr, bad flag
- `charge_template_ge68` — Ge68 charge template (TH3F)
- Calibration constants: `GeEvis`, `PDE`, `ESF`, `saturation`
- (If FCN evolves: time PDF, charge PDF, dark noise model — all in the struct)
**Output**: `double` (-2 ln L). No SNiPER dependency.
**Status**: ✅

### 1.3 E2E Test: ESD → Reco Vertex

```
Input:  ESD file (calibrated hit t/q, SiPM channel data)
        + CondDB calibration + bad channel list + Ge68 template
Output: reco vertex (x, y, z, energy) per event

This is a FULL pipeline test — runs the complete SNiPER algorithm.
```

**Golden reference**: `reference/ref_5evt.root`
- Generated by pimin's unmodified script, 2025-05-28
- 5 events from run 1427 file 001
- Immutable — never regenerate

**Test**: `tests/test_consistency.py`
- Runs reconstruction via minimal overlay
- Compares `TTree::Scan` output → must be **tree-identical**

**Status**: ✅

### 1.4 FCN Fixture — Capture ALL FCN Inputs + Output

```
Capture from pimin's UNMODIFIED code, for event 0 (first after warmup):

  Input (everything the FCN reads):
    - fChannelHit[8048]          calibrated charge per channel
    - bad_channel_list[8048]     which channels to skip
    - rPDE_list[8048]            relative PDE per channel
    - dcr_list[8048]             dark count rate per channel
    - channel_vec[8048]          SiPM positions (TVector3 → x/y/z)
    - charge_template_ge68       Ge68 TH3F (saved as filename reference)
    - GeEvis, PDE, ESF, saturation   calibration constants
    - Evis, vr, vtheta, vphi     fit parameters at initial guess

  Output:
    - FCN value (-2 ln L)        single double
```

**Method**: Temporarily instrument pimin's unmodified `QMLERec::QMLE()` with
`fprintf(stderr, ...)`, build from git commit `0ac33d6`, run 1 event, capture
stderr → `fixtures/fcn_evt0.txt`.

**Status**: ❌

### 1.5 FCN Unit Test — No Fitting

```
Input:  Load fixtures/fcn_evt0.txt → populate QMLEInput
        Call ComputeQMLE(input, Evis, vr, vtheta, vphi)  ← pure function, no Minuit
Output: Compare return value to golden FCN value
        Tolerance: ≤1e-13 relative drift
```

**Test**: `build/bin/test_qmle_fcn` (already exists as smoke test — extend to load fixture)
- Links against `libQMLEFCN.a` only — no SNiPER, no CondDB, no EDM
- Fast: runs in milliseconds
- Catches FP drift from innocent-looking changes

This is the FCN-level counterpart to 1.3 (E2E). Together they form the test-bed:
- 1.3: full pipeline, slow but comprehensive
- 1.5: FCN-only, fast, catches numerical regressions

**Status**: ❌ (blocked by 1.4)
