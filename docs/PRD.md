# TAO Reconstruction Speedup — PRD

## 1. Context
TAO reconstruction via TAOSW. Reference: omilrecv2 (JUNO OMILREC, 1525→189 ms/evt, 8.1x).

## 2. Goal
≥1.5x single-thread speedup, no resolution degradation.

## 3. Phase Overview

| Step | What | Deliverable |
|------|------|-------------|
| **1**: Test bed | Build + extract FCN + 2 golden tests | E2E test + FCN fixture test both pass |
| **2**: Profile | perf/flamegraph/phase-timing | Hotspot ranking |
| **3**: Optimize | One commit per opt, measure speed+drift | Speed improves, both tests still pass |
| **4**: Benchmark | 100-event timing, speed.csv | ms/evt per commit |
| **5**: Iterate | 2→3→4 loop | ≥1.5x reached |
| **6**: Acceptance | Resolution vs pimin baseline | No degradation |

---

## 4. Step 1: Test Bed — Detailed

Step 1 has four sub-steps. **1.2 (Refactor) must complete before 1.4 (FCN fixture)**
because the fixture test calls the FCN as a standalone free function.

```
Step 1:
  1.1 Build         → libRecQMLEAlg.so + libQMLEFCN.a
  1.2 Refactor      → extract QMLERec::QMLE() as free function ComputeQMLE()
  1.3 E2E test      → golden reference: ESD → reco vertex (x,y,z,E)
  1.4 FCN fixture   → golden fixture:  calibrated hits → FCN value (-2lnL)
  1.5 Gate          → both 1.3 and 1.4 must pass before any optimization
```

### 1.1 Build

Build `libRecQMLEAlg.so` (full SNiPER algorithm) and `libQMLEFCN.a` (standalone FCN, no SNiPER dependency) against CVMFS J26.1.0 + pimin's TAOSW overlay.

**Status**: ✅

### 1.2 Refactor — Extract FCN as Free Function

**Pre-requisite for 1.4.** Can't capture FCN fixture without a standalone FCN.

Extract from `QMLERec::QMLE(Evis, vr, vtheta, vphi)` → `ComputeQMLE(input, Evis, vr, vtheta, vphi)`:

- **Input struct** `QMLEInput`:
  - `fChannelHit[8048]` — calibrated charge per channel
  - `channels[8048]` — per-channel: position (flat x/y/z), rPDE, dcr, bad flag
  - `charge_template_ge68` — Ge68 TH3F template pointer
  - Calibration: `GeEvis`, `PDE`, `ESF`, `saturation`
- **Output**: `double` = -2 ln L
- **No SNiPER/EDM dependency** — ROOT-only (TVector3, TH3F)
- **Unit testable**: `build/bin/test_qmle_fcn` links against `libQMLEFCN.a`

**Status**: ✅ `ComputeQMLE` in `QMLEFCN.h/cc`, `QMLEInput` struct, 4 smoke tests pass.

### 1.3 E2E Test: ESD → Reco Vertex

```
Input:  ESD file (calibrated hit t/q, SiPM channel data)
Output: reco vertex (x, y, z, energy) per event
```

**Golden reference**: `reference/ref_5evt.root`
- Generated by pimin's unmodified script, 2025-05-28
- 5 events from run 1427 file 001
- Immutable — never regenerate

**Test**: `tests/test_consistency.py`
- Runs reconstruction via minimal overlay
- Compares `TTree::Scan` output → must be **tree-identical**

**Status**: ✅ Our code produces tree-identical output.

### 1.4 FCN Fixture: Calibrated Hits → FCN Value

```
Input:  QMLEInput (calibrated hits, channel geometry, PDE/DCR, template)
        + fit parameters (Evis, vr, vtheta, vphi) at initial guess
Output: double (-2 ln L)
```

**Golden fixture**: `fixtures/fcn_evt0.txt`
- Captured from pimin's **unmodified** code (commit `0ac33d6`)
- Instrument with `fprintf(stderr, ...)` in `QMLERec::QMLE()`, run 1 event
- Contains: Evis/vr/vtheta/vphi, per-channel (hit, bad, rPDE, dcr, x, y, z), FCN output

**Test**: `build/bin/test_qmle_fcn`
- Load fixture, populate `QMLEInput`, call `ComputeQMLE()`
- Compare to golden FCN value
- Tolerance: ≤1e-13 relative drift

**Status**: ❌
- `ComputeQMLE` extracted ✅
- Fixture NOT captured ❌
- Fixture test NOT written ❌

### 1.5 Gate: Both Tests Must Pass

Before any optimization commit (Step 3), both must pass:
1. `python tests/test_consistency.py` → PASS (tree-identical)
2. `build/bin/test_qmle_fcn` → PASS (FCN matches golden)

## 5. Step 2: Profile

- Phase timing: stopwatches in `QMLERec::execute()` (load, charge center, minimizer, output)
- `perf record -g` + FlameGraph
- Hotspot ranking: top 10 by self-time

---

## 6. Step 3: Optimize

- One optimization per commit
- Run 1.5 gate after each
- Speed regression ≥5% → rollback
- FCN drift >1e-13 → rollback or document

---

## 7. Step 4: Benchmark

- 100 events (after 10 warmup), single thread, Release
- Measure: Vertex Fit Time from per-event `TStopwatch`
- Append `benchmarks/speed.csv`

---

## 8. Steps 5-6: Iterate + Acceptance

Loop 2→3→4 until ≥1.5x. Acceptance: resolution comparison vs pimin on ≥1000 events.
