# TAO Reconstruction Speedup — PRD

## 1. Context
TAO reconstruction via TAOSW. Reference project: omilrecv2 (JUNO OMILREC, 1525→189 ms/evt, 8.1x).

## 2. Goal
≥1.5x single-thread speedup, no resolution degradation.

## 3. Phase Overview

| Step | What | Deliverable |
|------|------|-------------|
| **1**: Test bed | Build + extract FCN + E2E test + FCN fixture | Two golden tests: E2E and FCN |
| **2**: Profile | perf/flamegraph/phase-timing | Hotspot ranking |
| **3**: Optimize | One commit per opt, measure speed+drift | Speed improves, tests pass |
| **4**: Benchmark | 100-event timing, speed.csv | ms/evt per commit |
| **5**: Iterate | Step 2→3→4 loop until goal | ≥1.5x |
| **6**: Acceptance | Resolution comparison vs pimin baseline | No degradation |

## 4. Step 1: Test Bed

Step 1 has four sub-steps: build, refactor (pre-requisite for fixture), E2E, and FCN fixture.

### 4.0 Build
- cmake + make → `libRecQMLEAlg.so` (against CVMFS J26.1.0 + pimin's TAOSW overlay)
- `libQMLEFCN.a` — static lib for standalone FCN testing (no SNiPER dependency)

### 4.0b Refactor — Extract FCN as free function
**Why**: 1b requires a standalone FCN to test. Must be done before 1b.

- Extract `QMLERec::QMLE()` → `ComputeQMLE(QMLEInput, Evis, vr, vtheta, vphi)` in `QMLEFCN.h/cc`
- Dependencies: `ChargeTemplate` (TH3F interpolation), `TVector3` (geometry)
- No SNiPER/EDM dependency — callable from unit tests
- Flatten `QMLEChannelData`: TVector3 → posX/Y/Z (cache-friendly)
- `QMLEInput` struct holds: `fChannelHit[8048]`, channel data, template pointer, calibration constants

### 4.1a E2E Test: ESD → Reco Vertex

```
Input:  ESD file (calibrated hit t/q, SiPM channel data)
Output: Reco vertex (x, y, z, energy)
```

**Golden reference**: `reference/ref_5evt.root`
- Generated by pimin's unmodified script on 2025-05-28
- 5 events from run 1427 file 001
- MUST NOT be regenerated

**Test**: `tests/test_consistency.py`
- Runs reconstruction via minimal overlay (only our `libRecQMLEAlg.so`, no SNiPER dupes)
- Compares TTree::Scan output to golden — must be **tree-identical**

**Status**: ✅ DONE. Our optimized code produces tree-identical output to the golden.

### 4.1b FCN Fixture: Calibrated Hits → FCN Value

```
Input:  fChannelHit[8048], bad_channel_list[8048], rPDE_list[8048], dcr_list[8048],
        channel_vec[8048], charge_template_ge68,
        fit parameters (Evis, vr, vtheta, vphi),
        calibration constants (GeEvis, PDE, ESF, saturation)
Output: -2 ln L (single double)
```

**How to capture**:
1. Build pimin's unmodified code from git (commit `0ac33d6`)
2. Add temporary `fprintf(stderr, ...)` instrumentation to `QMLERec::QMLE()`
3. Run 1 event, capture fixture lines from stderr → `fixtures/fcn_evt0.txt`
4. Revert to clean code; fixture file becomes the golden

**Test**: `build/bin/test_qmle_fcn`
- Load fixture, call `ComputeQMLE()`, compare to golden value
- Tolerance: ≤1e-13 relative drift

**Status**: ❌ NOT DONE
- `ComputeQMLE` free function already extracted (4.0b) ✅
- Fixture NOT captured from pimin baseline ❌
- Fixture test NOT built ❌

## 6. Step 3: Optimize

Rules:
- One optimization per commit
- Run both tests (1c) after each
- Run benchmark after each meaningful change
- Speed regression ≥5% → rollback
- FCN drift >1e-13 → rollback or document

Optimization categories:
- Hoist invariants out of Minuit loop
- Data layout (SoA, cache-friendly)
- Algebraic (remove exp/log round-trips, fast math)
- SIMD vectorization
- Algorithmic (reduce iteration count, spatial indexing)

## 7. Step 4: Benchmark

- 100 events minimum (after 10-event warmup)
- Measure: Vertex Fit Time (per-event stopwatch in QMLERec)
- Append to `benchmarks/speed.csv`: commit, ms/evt, date
- Speed gate: merge request fails if speed regresses ≥5% vs main
