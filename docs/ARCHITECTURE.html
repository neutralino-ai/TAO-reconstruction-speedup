1. The architecture in one picture

The current design splits the old monolithic reconstruction into a SNiPER-facing orchestration layer and a standalone numerical FCN layer. The key improvement is that QMLERec::QMLE() now adapts SNiPER state into QMLEInput and delegates the likelihood to ComputeQMLE().

  └─ links QMLEFCN only, not SNiPER

The standalone QMLEFCN target is the testability seam: fast, deterministic, and independent from SNiPER event services.

4. Runtime data flow

CdCalibEvt channels→ fChannelHit[8048]→ rPDE / dcr / bad flags→ channel_vec geometry→ QMLEInput→ ComputeQMLE→ Minuit objective

The fit parameters Evis, vr, vtheta, and vphi are not outputs of the FCN. They are trial inputs supplied by Minuit. The FCN output is a single scalar likelihood value.

6. Charge template role

ChargeTemplate is a wrapper around a Ge68 ROOT template. In practice the physics lookup is the ROOT object:

RecQMLEAlg/input/Ge68_charge_template.root
  └─ h3_interpolation : TH3F

For each trial vertex, it interpolates expected charge using radius, SiPM angle, and vertex polar angle.

8. Dependency direction

Healthy direction is top-down: SNiPER depends on FCN, tests depend on FCN/SNiPER, but FCN does not depend on SNiPER.

9. SOLID-style design health

PrincipleScoreCurrent reading
Single Responsibility8/10
QMLEFCN is focused. QMLERec still does many SNiPER orchestration tasks, which is expected for an algorithm class.
Open/Closed7/10
The FCN seam allows optimization variants without rewriting E2E code. More strategy interfaces could help later.
Liskov6/10
Little inheritance in new code; SNiPER/Minuit base classes are external constraints.
Interface Segregation7/10
QMLEInput is explicit but still large. Good for fixture completeness; could be decomposed later.
Dependency Inversion7/10
FCN avoids SNiPER dependency. It still directly depends on ChargeTemplate/ROOT interpolation.

10. What is clean now

  • ✓ SNiPER path and FCN test use the same ComputeQMLE().
  • ✓ FCN fixture captures all per-channel inputs needed by the likelihood.
  • ✓ Unit test is fast and independent of CondDB/SNiPER.
  • ✓ E2E test protects full-pipeline behavior against pimin golden output.

11. Design debt to watch

  • ! QMLERec still owns many responsibilities: event I/O, calibration adaptation, fit setup, output, and fixture capture.
  • ! ChargeTemplate is a direct ROOT dependency inside the FCN layer.
  • ! QMLEInput is intentionally complete but large; adding sub-structs may improve clarity later.
  • ! QMLERec::CalExpChargeHit() duplicates expected-charge scaling used for output bookkeeping.
